## 執行方法
##### 一、啟動系統流程
- 確認已安裝 Python 3.11.5 環境，並下載本專案程式碼。
- 於專案資料夾中，使用終端機執行以下指令啟動API伺服器：
`python.exe Final0108\\final0108.py --serve`

##### 二、進入系統方式
- 啟動成功後，於瀏覽器輸入網址：http://127.0.0.1:8000/ 即可進入本系統操作介面。

##### 三、操作方式
- 在欄位中輸入欲查詢之**股票代碼**、**基本資料**與**策略參數**，系統將自動抓取股價資料並進行技術指標分析。
- ⚠️查詢股票代碼若為台股或其他國家股票則需另外標記（如：台積電為 2330.tw ）。


##### 四、系統輸出結果
- 系統會根據使用者的策略描述，顯示MDD(最大回撤)、勝率、盈虧、總報酬，並提供資金曲線圖與近十筆交易資料，同時依據策略條件判斷進場或出場訊號，輔助使用者決策。

## 專案設計說明

#### 我們這組的策略為：
1. 透過長短期均線（SMA）判斷中長期趨勢。
2. 相對強弱指標（RSI）用於判斷市場動能與超買超賣區間，作為進出場輔助依據（突破時買進，轉弱時賣出）。
3. 最後用成交量做進一步確認。

  * **進場時機**
    短SMA > 長SMA （黃金交叉），且RSI > 進場門檻，或RSI正在上升。

  * **平倉時機**
短SMA < 長SMA （死亡交叉），且RSI存在顯著回落。

#### 專案架構：
- `scrpts - render_index.py`：輔助腳本，用於生成靜態索引或報表
- `static - style.css`：網頁頁面的CSS設計
- `README.MD`：專案摘要與使用說明
- `final0108.csv`：回測的輸出資料
- `final0108.py`：主程式，完整流程為：__依賴檢查 → 下載資料 → 計算指標 → 產生訊號 → 回測 → 產生報表與 FastAPI 網頁/API__
- `final0108_design.md`：專案設計說明
- `pyproject.html`：Python 專案元資料
- `requirements.txt`：相依套件清單（pip install 可用）

## 心得總結

我們在這次專案中完成了一個可以根據自己策略完成回測的網站，雖然程式方面都是由AI完成的，但在製作過程中還是有部分發現與遇到困難的部分。

策略的部分沒有思考太久，主要是以黃金交叉/死亡交叉搭配其他指標決定買進或賣出，不算過於複雜的策略，所以交給AI執行程式算是滿順利的，檢查過後也發現網頁能順利如期運行。

在網頁製作與網頁設計的部分卡關了一段時間，對於第一次製作網頁的我們來說，在這部分沒有太多概念，也不太清楚怎麼建立網頁，最後是透過查詢相關資料與詢問ChatGPT慢慢了解後才將其完成；網頁設計則是常常不符合預期的樣子，過程中我們也從中發現要給出夠清楚、明確的指示給AI執行才會有較高的成功機率，經過反覆修改後才完成。

最後是上傳至GitHub的部分，由於之前沒有操作過，所以在這部分也花了點時間，途中也有遇到檔案跑掉、沒上傳到等情況，最後試了幾次加上看了幾篇文章後才成功上傳。

本次策略網頁的回測結果分析分成三部分，分別為回測結果、資金曲線圖和近十筆交易。回測結果包含了作業要求的MDD、勝率和盈虧比以及總報酬，其中MDD代表回測期間內資產從最高點跌至最低點經歷的最大虧損幅度，為面臨的最大風險；我們主要以RSI 55進場 45出場、短期SMA採20日、長期60日的情況進行回測，發現勝率、盈虧比、總報酬通常為正，但MDD的存在也表示策略存在風險，代表使用此策略的同時也需要另外進行資金控管，不能只依賴單一策略進行操作。

經過這次的期末專案，我們了解到一個網頁是需要經過多個程式與檔案運作的，不管是程式本身還是網頁設計都需要花費不少心思調整、設計；在現今環境中，如何正確的提供指示給AI完成需求也是一大學問，透過AI提供程式讓網頁的架設變得更容易。 除了網頁本身，我們在這次製作中對於某些技術指標也有了新的認識（如MDD、RSI…等等），此外，也了解到老師提過的財務領域人員要是也擁有程式方面的能力在產業內會很吃香的部分。本次專案讓我們體會到程式能力、財務知識與 AI 工具結合的重要性，對未來在財務相關領域的學習與發展具有相當大的啟發。
